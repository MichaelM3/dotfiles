# Verify Plan

After drafting a plan, re-read it against repo evidence.

Checklist:

- List assumptions.
- Search files needed to prove or disprove assumptions.
- Revise plan if repo evidence conflicts.
- Ask user only for product/strategy blockers.
- End with `Plan verified` or `Plan needs revision`.
